# Консольная игра с системой контрольных точек (Паттерн Memento)

Прикладная проблема: Игроки часто совершают ошибки, из-за этого персонажи могут погибнуть или попадают в невыгодные ситуации. Без возможности сохранения прогресса игра становится слишком хардкорной и это может не понравится некоторым игрокам. Для этого необходима надежная система контрольных точек, позволяющая:
Сохранять текущее состояние игрока (здоровье, уровень, инвентарь) перед опасными зонами;
Автоматически или вручную восстанавливать это состояние в случае гибели персонажа;
Предотвращать прямое изменение (порчу) сохранённых данных со стороны игровых систем.

Для решения этой задачи в приложении реализован паттерн проектирования **Memento**.

### Цель паттерна
Зафиксировать и сохранить внутреннее состояние объекта (игрока) так, чтобы позже объект можно было восстановить в это же состояние.

### Какую задачу решает паттерн Memento?
**Изоляция данных:** Класс, управляющий историей сохранений (CheckpointManager), не имеет прямого доступа к полям здоровья или оружия игрока. Он оперирует лишь объектами снимков;
**История и откат:** Паттерн позволяет легко организовать стек контрольных точек, реализуя механику последовательного отката назад во времени;
**Упрощение кода игрока:** Сам персонаж (Player) не занимается менеджментом файлов или стека сохранений, он умеет только создавать снимок себя и восстанавливаться по нему.

### Архитектура паттерна:
1. Player - объект, состояние которого нужно сохранять;
2. PlayerMemento - неизменяемый объект, хранящий копию характеристик игрока;
3. CheckpointManager - опекун, который хранит историю снимков в стеке.
