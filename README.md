# Консольная игра с системой контрольных точек (Паттерн Memento)

**Прикладная проблема**: Игроки часто совершают ошибки, из-за этого персонажи могут погибнуть или попадают в невыгодные ситуации. Без возможности сохранения прогресса игра становится слишком хардкорной и это может не понравится некоторым игрокам. Для этого необходима надежная система контрольных точек, позволяющая:

1. Сохранять текущее состояние игрока (здоровье, уровень, инвентарь) перед опасными зонами;
2. Автоматически или вручную восстанавливать это состояние в случае гибели персонажа;
3. Предотвращать прямое изменение (порчу) сохранённых данных со стороны игровых систем.

Для решения этой задачи в приложении реализован паттерн проектирования **Memento**.

## Цель паттерна
Зафиксировать и сохранить внутреннее состояние объекта (игрока) так, чтобы позже объект можно было восстановить в это же состояние.

## Какую задачу решает паттерн Memento?

+ **Изоляция данных:** Класс, управляющий историей сохранений (CheckpointManager), не имеет прямого доступа к полям здоровья или оружия игрока. Он оперирует объектами снимков;
+ **История и бесконечные загрузки:** Паттерн с двусвязным списком позволяет игроку загружать последнюю сохраненную точку бесконечное количество раз.
+ **Упрощение кода игрока:** Сам персонаж (Player) умеет только создавать снимок себя и восстанавливаться по нему.

## Архитектура:

1. **Player** - объект, состояние которого нужно сохранять;
2. **PlayerMemento** - неизменяемый объект, хранящий копию характеристик игрока;
3. **CheckpointManager** - хранит историю снимков;
4. **GameLogger** - выводит ввод/вывод в консоль
